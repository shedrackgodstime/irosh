# Honest Review: irosh v0.1.0

**What's Good:**
- Clean modular architecture with clear separation (transport, storage, session, client, server)
- Feature flags done right - library stays lean for embeddings
- TOFU trust model is idiomatic for P2P auth
- Error hierarchy is well-designed with `thiserror`
- Documentation structure is solid with architecture/security docs
- Proper re-export of russh for consumers

**Areas to Improve:**

1. **Feature flag bug** in `Cargo.toml:23-24`:
   ```toml
   server = [...dep:russh", "dep:portable-pty"]
   client = [...dep:russh", "dep:portable-pty"]
   ```
   The CLI enables both features - so portable-pty gets compiled even for client-only use. These should be separate.

2. **Test coverage is thin** - there's `mod tests` files but few actual test cases. You have the structure, now add integration tests.

3. **Missing from v0.1.0:**
   - No connection timeout handling
   - No reconnection logic
   - No graceful disconnect protocol between peers
   - Transfer resumption not supported
   - No docs for the CLI commands

4. **Public API polish:**
   - `ServerReady` exposes internal transport types in return signatures
   - `IroshError` is `#[non_exhaustive]` but you likely want that stable

5. **CI/testing:** No automated tests in workflows yet

**Verdict:** Solid foundation for v0.1.0. The architecture will hold as you scale. Priority for next release: fix the feature flags, add connection timeouts, and more tests.
